# Cloud Features Part 3 — File Sharing

**Date:** 2026-04-04  
**Commit:** feat: file sharing with expiring public download links  

---

## Overview

Adds a secure, expiring public file-sharing system to the docpilot file manager. Any user can generate a temporary public download link for a file; the link works without any login, making it ideal for sharing with clients or external teams.

---

## Backend Changes (`routes/fileRoutes.js`)

### New: Share Link System

**Data store:** `src/DataFiles/shares.json`

```json
{
  "shares": {
    "<shareId>": {
      "project": "Laich-Suppingen",
      "filePath": "Doku/SUPPN/APL/report.pdf",
      "fileName": "report.pdf",
      "createdBy": "rishi@geggos.app",
      "createdAt": "2026-04-04T14:00:00Z",
      "expiresAt": "2026-04-11T14:00:00Z",
      "accessCount": 0
    }
  }
}
```

- `shares.json` is auto-created if it doesn't exist.
- Expired entries are lazily cleaned on every share endpoint call.

### New Endpoints

| Method | Route | Auth | Description |
|--------|-------|------|-------------|
| `POST` | `/api/files/share?project=X` | Admin only | Create a share link |
| `GET` | `/api/files/shares?project=X&filePath=Y` | Any user | List active shares for a file |
| `DELETE` | `/api/files/share?project=X` | Admin only | Revoke a share link |
| `GET` | `/share/:shareId` | **Public (no auth)** | Download the shared file |

### Share Token Generation

Uses `crypto.randomBytes(9).toString('base64url')` — cryptographically secure 12-character token.

### Public Download Endpoint (`serveShare`)

- Exported as `module.exports.serveShare` and mounted in `server.js` before all auth middleware
- Returns clean HTML error pages for expired (HTTP 410) or missing (HTTP 404) links
- Increments `accessCount` on each successful download
- `expiresIn` defaults to 168 hours (7 days), max 720 hours (30 days)

## Server Changes (`server.js`)

Mounted the public share route before auth middleware:

```js
const { serveShare } = require('./routes/fileRoutes');
app.get('/share/:shareId', serveShare);
```

## Frontend Changes (`files.html`)

### Context Menu

Added **Share** option for files (not folders) between Download and the Details divider:

```
Download
Share    ← NEW (emerald link icon)
─────────
Details
...
```

Share is available to **all users** (not just admins). Generating/revoking links is admin-only at the API level; viewing and copying links works for everyone.

### Share Modal

Opens when "Share" is clicked in the context menu. Adapts based on share state:

**No active share:**
- Expiry dropdown (1 day / 3 days / 7 days / 30 days)
- "Generate Link" button

**Active share exists:**
- Full URL display (monospace, read-only input)
- "Copy Link" button with 2s checkmark feedback
- Expiry info ("Expires in X days")
- Download count ("Downloaded X times")
- "Revoke Link" button (confirms before deletion)
- "Generate New Link" button (revokes existing, creates fresh one)

### New Icon

Added `share` icon (emerald chain-link) to `iconSVGInline()`.

## Security Notes

- Share tokens are cryptographically random (96 bits of entropy)
- Path traversal protection via `safePath()` on all file access
- Public endpoint does not expose any project structure or auth headers
- Expired links return HTTP 410 Gone (not 404) — semantically correct
- Admin-only creation/revocation enforced via `x-user-role` header check
