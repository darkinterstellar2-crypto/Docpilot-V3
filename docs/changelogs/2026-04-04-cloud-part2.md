## What's New

### Folder Picker Modal

A new modal for selecting a destination folder when copying or moving files/folders.

- **Design:** Matches the existing glassmorphism style — blurred backdrop, rounded white modal box, same fonts and colors
- **Tree view:** Fetches the full folder tree from `GET /api/files/tree?project=X` on open
- **Expand/collapse:** Chevron arrows toggle folder children; root level auto-expands on load
- **Selection:** Click a folder row to highlight it as the destination; selected path shown below the tree
- **Create New Folder:** Inline input + Create button; press Enter to confirm. Creates folder at the selected path (or root if none selected), then refreshes the tree
- **Confirm button:** Dynamically labeled "Copy Here" or "Move Here" depending on mode
- **Cancel button:** Closes the modal without action
- **Error display:** API/network errors shown inline inside the modal (modal stays open)
- **Disabled states:** Can't select the source item's current directory as a move target; can't move/copy a folder into itself or a subdirectory of itself
- **Mobile-friendly:** Responsive width (`90vw`, max `520px`)

### Context Menu — Copy to… and Move to…

Two new admin-only entries added between **Details** and **Rename**:

```
Open / Download
─────────────
Details
─────────────
Copy to…        ← NEW
Move to…        ← NEW
─────────────
Rename
Move to Recycle Bin
```

- Both entries open the folder picker in the appropriate mode
- After a successful operation: modal closes, file list reloads, success toast shown
- Both are guarded by the existing `isAdmin` check

### Copy / Move Implementation

- **Copy:** `POST /api/files/copy?project=X` with `{ source, destination }`
- **Move:** `POST /api/files/move?project=X` with `{ source, destination }`
- Headers sent: `x-user-email`, `x-user-role`, `Content-Type: application/json`
- `source` is built as `currentPath/itemName` (relative from project root)
- `destination` is the folder path selected in the picker
- Loading state on confirm button while request is in flight
- Success → close modal, reload file list, brief toast notification
- Error → error message shown in modal, button re-enabled
