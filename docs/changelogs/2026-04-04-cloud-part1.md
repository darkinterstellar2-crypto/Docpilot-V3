## New Endpoints

### POST /api/files/copy

Copy a file or folder to a new location within the same project.

**Query parameter:** `?project=<ProjectName>`

**Request body:**
```json
{
  "source": "SUPPN/APL/report.pdf",
  "destination": "LAICH/APL/"
}
```

**Success response:**
```json
{ "success": true, "destination": "LAICH/APL/report.pdf" }
```

**Behavior:**
- Accepts a relative `source` path and a relative `destination` directory, both within the project root.
- If source is a **file**: copied into the destination directory, keeping the original filename.
- If source is a **folder**: entire subtree copied recursively.
- **Conflict resolution**: if a file/folder with the same name already exists at the destination, it is auto-renamed (`report (1).pdf`, `report (2).pdf`, …).
- Destination directory is created automatically if it doesn't exist.
- `fileMeta` updated for every copied file (marks `modifiedBy`/`modifiedAt` for the requesting user).
- Action logged via `logAction()`.
- **Admin-only** — requires `x-user-role: admin | administrator | superadmin` header.

**Error cases:**
| Status | Condition |
|--------|-----------|
| 400 | Missing `project`, `source`, or `destination` |
| 400 | Invalid path (traversal detected) |
| 400 | Source and destination resolve to the same path |
| 403 | Non-admin user |
| 404 | Source does not exist |
| 500 | Filesystem error |

### POST /api/files/move

Move (rename) a file or folder to a new location within the same project.

**Query parameter:** `?project=<ProjectName>`

**Request body:**
```json
{
  "source": "SUPPN/APL/report.pdf",
  "destination": "LAICH/APL/"
}
```

**Success response:**
```json
{ "success": true, "destination": "LAICH/APL/report.pdf" }
```

**Behavior:**
- Same path validation and conflict-resolution logic as `/copy`.
- Uses `fs.rename()` — atomic on same filesystem, falls back to OS behaviour across mounts.
- **fileMeta** updated: old relative-path keys renamed to new keys (including all child entries for folder moves), and the destination marked with current user.
- Prevents moving a folder into itself (`destination` must not start with `source + sep`).
- Action logged via `logAction()`.
- **Admin-only** — same header requirement as `/copy`.

**Error cases:** identical to `/copy`, plus:
| Status | Condition |
|--------|-----------|
| 400 | Destination is inside source (folder-into-itself) |

### GET /api/files/tree

Return the full directory tree of a project (directories only — no files). Used by the folder-picker UI for copy/move target selection.

**Query parameter:** `?project=<ProjectName>`

**Success response:**
```json
{
  "success": true,
  "tree": {
    "name": "Root",
    "path": "",
    "children": [
      {
        "name": "Doku",
        "path": "Doku",
        "children": [
          { "name": "Aufmass", "path": "Doku/Aufmass", "children": [] },
          { "name": "SUPPN",   "path": "Doku/SUPPN",   "children": [] }
        ]
      },
      { "name": "Pläne", "path": "Pläne", "children": [] }
    ]
  }
}
```

**Behavior:**
- Recursively scans the project root for **directories only**.
- **Excluded** from the tree:
  - Directories starting with `.` (hidden), including `.trash`
  - The `chat` directory
- Maximum traversal depth: **10 levels** (prevents runaway recursion on deep trees).
- Directories sorted **alphabetically** at each level.
- Paths are relative to the project root (forward-slash separated).

**Error cases:**
| Status | Condition |
|--------|-----------|
| 400 | Missing `project` |
| 404 | Project root not found on disk |
| 500 | Filesystem error |

## Helper Functions Added (internal)

### `getAutoRenamedName(destDir, baseName)`
Finds the next free filename in `destDir`. Returns `baseName` if free, otherwise `stem (1).ext`, `stem (2).ext`, etc.

### `copyRecursive(src, dest, projectRoot, projectName, userEmail)`
Recursively copies a file or directory tree from `src` to `dest`. Calls `setFileMeta()` for every copied file to track the modifying user.

### `buildDirTree(dirPath, projectRoot, depth, maxDepth)`
Recursive directory-only tree builder used by `/tree`. Respects depth limit and exclusion rules.

### `isAdmin(req)`
Returns `true` if `x-user-role` header is `admin`, `administrator`, or `superadmin` (case-insensitive).

## Security Notes

- All user-supplied paths validated through `safePath()` before any filesystem access — path traversal is blocked at the boundary.
- Copy and Move are admin-only operations.
- `source === destination` and folder-into-itself cases are explicitly rejected with `400`.
